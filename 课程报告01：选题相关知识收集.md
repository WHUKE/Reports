# 课程报告01：选题相关知识收集

## 选题

**职业规划知识图谱** — 基于招聘岗位描述（JD）构建职业规划领域知识图谱，支持职位推荐、就业技能成长规划、就业决策等应用场景。

## 拟完成的任务

本项目围绕"职业规划"主题，拟完成以下任务：

1. **知识图谱构建**：从大量 JD 中抽取职位、技能、学历、经验等实体及关系，构建职业规划领域知识图谱
2. **职位推荐**：基于用户技能画像，在图谱上匹配推荐适合的职位
3. **技能成长规划**：分析职位间的技能差距，为用户生成从当前岗位到目标岗位的技能学习路径
4. **就业决策支持**：通过图谱提供岗位对比、行业趋势等辅助决策信息

## 数据源

### 主要数据源：招聘岗位描述（JD）

| 项目 | 说明 |
|------|------|
| 来源 | 国内互联网大厂官方招聘页面（手动收集） |
| 采集方式 | 小组成员每人手动收集 20 条 JD，纯文本格式，跑通demo后将使用爬虫批量获取 |
| 当前规模 | 共 120 条 JD（6人 × 20条），存储于 `data/raw/` 目录 |
| 数据格式 | TXT 纯文本，包含职位名称、工作职责、任职要求、加分项等 |
| 岗位类型 | 覆盖后端开发、前端开发、数据分析、AI/机器学习、运维、爬虫、游戏引擎等技术方向 |
| 后续扩展 | Demo 跑通后考虑通过爬虫增量采集更多 JD |

### 候选补充数据源

- **O\*NET（美国职业信息网络）**：全球最大的职业信息知识库，包含职业分类、技能要求、工作活动等结构化数据，可作为图谱 Schema 设计参考
- **中国职业分类大典**：国内官方职业分类标准，可用于职位类别归一化
- **招聘平台公开数据**：如 Boss直聘、猎聘等平台的公开岗位信息，用于扩大数据规模

## 知识收集与构建方法

### 整体架构

```
原始JD (TXT) → 正则预解析 → LLM深度抽取 → 技能归一化 → 结构化JSON → 知识图谱
```

### 具体方法

#### 1. 正则预解析

使用基于正则表达式的规则解析器，从 JD 文本中提取格式化元信息：
- 职位名称、工作地点、学历要求、工作年限
- 招聘类型、所属部门、职位类别
- 工作职责列表、任职要求列表

特点：格式无关设计，支持多种 JD 文本格式，对每个字段使用多种宽松模式逐个匹配。

#### 2. LLM 深度抽取

使用 DeepSeek 大语言模型对 JD 文本进行深层知识抽取：
- **细粒度技能拆分**：将"熟悉 MySQL/Redis/MongoDB"拆分为三个独立技能实体
- **技能熟练度识别**：提取每个技能的要求级别（了解/熟悉/熟练/精通）
- **技能分类标注**：自动标注技能类别（编程语言/框架/数据库/DevOps工具/软技能等）
- **正则+LLM互补**：正则负责结构化元信息，LLM负责语义级技能抽取

#### 3. 技能名称归一化

为确保知识图谱中同一技能只有一个节点，建立了 150+ 条技能别名映射表：
- 缩写归一化：`JS` → `JavaScript`，`K8S` → `Kubernetes`
- 版本归一化：`Vue3` / `Vue 3` / `Vue2` → `Vue.js`
- 同义归一化：`Golang` → `Go`，`ES` → `Elasticsearch`
- 自动去重：同一 JD 中重复技能只保留一次

### 知识表示设计

面向知识图谱设计的结构化输出 Schema：

```json
{
  "source_file": "xyh_01.txt",
  "job_title": "全栈开发工程师",
  "location": "广州市",
  "education": "本科",
  "experience": "3-5年",
  "job_category": "前端开发",
  "responsibilities": ["职责1", "职责2"],
  "required_skills": [
    {"name": "JavaScript", "proficiency": "精通", "category": "编程语言"},
    {"name": "Vue.js", "proficiency": "精通", "category": "前端框架"},
    {"name": "MySQL", "proficiency": "熟悉", "category": "数据库"}
  ],
  "preferred_skills": [
    {"name": "Docker", "proficiency": "熟悉", "category": "DevOps工具"}
  ]
}
```

预期知识图谱实体与关系：

| 实体类型 | 示例 |
|----------|------|
| 职位 (Job) | 全栈开发工程师、后端开发工程师 |
| 技能 (Skill) | JavaScript、MySQL、Docker |
| 地点 (Location) | 广州市、上海、北京 |
| 学历 (Education) | 本科、硕士 |
| 职位类别 (Category) | 前端开发、后端开发、AI/机器学习 |

| 关系类型 | 说明 |
|----------|------|
| REQUIRES_SKILL | 职位 → 技能（附带熟练度） |
| PREFERS_SKILL | 职位 → 加分技能 |
| REQUIRES_EDUCATION | 职位 → 学历 |
| REQUIRES_EXPERIENCE | 职位 → 工作年限 |
| LOCATED_IN | 职位 → 地点 |
| BELONGS_TO | 职位 → 职位类别 |
